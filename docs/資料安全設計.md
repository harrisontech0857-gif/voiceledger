## 1. 概述

本文件概述了語記應用中個人財務數據的安全設計原則。目標是遵循 OWASP、GDPR 和台灣《個人資料保護法》的最佳實踐，確保用戶數據得到妥善保護。

### 1.1 安全目標

- **機密性**（Confidentiality）：防止未授權的數據訪問
- **完整性**（Integrity）：確保數據未被篡改
- **可用性**（Availability）：確保授權用戶可以訪問數據
- **問責性**（Accountability）：記錄所有數據訪問和操作

### 1.2 安全層級

| 層級 | 數據類型 | 加密 | 訪問控制 |
|------|---------|------|--------|
| **Tier 1** | 個人識別信息 (PII) | AES-256 | 嚴格 RLS |
| **Tier 2** | 財務交易 | 根據配置 | RLS + IP 限制 |
| **Tier 3** | 位置數據 | AES-256 | RLS + 時間限制 |
| **Tier 4** | 日誌和元數據 | 根據內容 | RLS + 審計記錄 |

## 2. 傳輸層安全

### 2.1 TLS 加密

所有客戶端到伺服器的通信都使用 TLS 1.3 進行加密。

**配置**：

```
最小 TLS 版本: 1.3
推薦 TLS 版本: 1.3
密碼套件: TLS_AES_256_GCM_SHA384 (首選)
             TLS_CHACHA20_POLY1305_SHA256
             TLS_AES_128_GCM_SHA256
```

### 2.2 證書管理

- **頒發機構**：Supabase/Let's Encrypt
- **效期**：90 天
- **自動更新**：啟用
- **備用域名**：dpqgrwoqalwfodblctqm.supabase.co
- **HSTS**：啟用（max-age: 31536000）

### 2.3 API 端點安全

所有 API 端點需要：

1. **有效的 HTTPS 連接**
2. **有效的身份認證令牌**（JWT）
3. **適當的 CORS 標題**
4. **速率限制**（防止暴力攻擊）

**速率限制規則**：

```
默認：100 請求/小時/IP
登錄：5 嘗試/5 分鐘/IP
資料導出：1 請求/24 小時/用戶
```

## 3. 靜態數據加密

### 3.1 數據庫級別加密

#### 個人身份識別信息 (PII)

**包含字段**：

- `users.email` → pgcrypto AES-256-CBC
- `users.phone_number` → pgcrypto AES-256-CBC
- `users.display_name` → 不加密（非敏感）

**加密函數**：

```sql
-- 加密
SELECT pgp_sym_encrypt('plain_text', 'symmetric_key'::bytea, 'cipher-algo=aes256');

-- 解密
SELECT pgp_sym_decrypt(encrypted_data, 'symmetric_key'::bytea)::TEXT;
```

**密鑰管理**：

- 主密鑰存儲在 Supabase Vault（環境變數）
- 密鑰輪換：每年一次
- 備份密鑰：安全存檔

#### 位置數據

**包含字段**：

- `geofence_logs.location_latitude`
- `geofence_logs.location_longitude`
- `photo_logs.extracted_text`（可能包含位置）

**加密方法**：同 PII

**保留期**：30 天（用戶可配置 7-90 天）

#### 支付信息

**包含字段**：

- `user_subscriptions.payment_method`
- `billing_records.transaction_id`

**加密方法**：AES-256

**PCI DSS 合規性**：不直接存儲信用卡詳情；使用 RevenueCat 或 Stripe 進行支付處理

### 3.2 應用層加密

不敏感字段（如交易描述）可以在應用層使用可選的端到端加密（E2E）。

**E2E 加密流程**（當用戶啟用時）：

```
用戶輸入 → 應用側加密 → 伺服器存儲密文 → 解密需要用戶的私鑰
```

## 4. 數據庫安全

### 4.1 Row Level Security (RLS)

所有敏感表都啟用 RLS：

```sql
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
ALTER TABLE transactions ENABLE ROW LEVEL SECURITY;
ALTER TABLE geofence_logs ENABLE ROW LEVEL SECURITY;
-- ... 其他表
```

**RLS 政策示例**：

```sql
-- 用戶只能看到自己的交易
CREATE POLICY "用戶只能看到自己的交易" ON transactions
  FOR SELECT USING (
    user_id = auth.uid()
    OR family_ledger_id IN (
      SELECT id FROM family_ledgers
      WHERE family_id IN (
        SELECT family_id FROM family_members WHERE user_id = auth.uid()
      )
    )
  );

-- 用戶只能創建自己的交易
CREATE POLICY "用戶只能創建自己的交易" ON transactions
  FOR INSERT WITH CHECK (user_id = auth.uid());
```

### 4.2 SQL 注入防護

- **參數化查詢**：所有查詢使用參數綁定
- **ORM 使用**：Supabase 的 PostgREST API 自動進行參數化
- **查詢審查**：定期審查 SQL 查詢的安全性

**不安全的查詢示例**（避免）：

```typescript
// 危險 - 不要這樣做
const query = `SELECT * FROM users WHERE id = '${userId}'`;
```

**安全的查詢示例**：

```typescript
// 安全 - 使用參數化
const { data } = await supabase
  .from('users')
  .select('*')
  .eq('id', userId);
```

### 4.3 備份和恢復

- **備份頻率**：每日自動備份
- **備份保留**：30 天
- **加密備份**：是（在傳輸和存儲時）
- **恢復測試**：每月進行一次
- **地理冗餘**：備份存儲在多個地區

## 5. 身份認證與授權

### 5.1 身份認證方法

應用支持多種身份認證方法：

| 方法 | 安全級別 | 使用場景 |
|------|---------|--------|
| **郵箱 + 密碼** | 基礎 | 初始註冊 |
| **Google OAuth 2.0** | 中等 | 快速登錄 |
| **Apple ID** | 中等 | iOS 用戶 |
| **多因素認證 (MFA)** | 高 | 可選啟用 |

### 5.2 密碼政策

- **最少長度**：12 字符
- **複雜度**：至少包含大寫、小寫、數字、特殊字符
- **密碼過期**：180 天（建議用戶更改）
- **歷史記錄**：禁止重複最近 5 個密碼
- **哈希算法**：bcrypt（Supabase 默認）

### 5.3 會話管理

- **令牌類型**：JWT（JSON Web Token）
- **訪問令牌壽命**：1 小時
- **刷新令牌壽命**：7 天
- **會話超時**：30 分鐘無活動後
- **令牌撤銷**：支持（用戶登出時）

**JWT 載荷示例**：

```json
{
  "sub": "user-uuid",
  "email": "user@example.com",
  "aud": "authenticated",
  "iat": 1234567890,
  "exp": 1234571490
}
```

### 5.4 多因素認證 (MFA)

可選的 MFA 支持：

- **TOTP**（時間基礎一次性密碼）
- **SMS**（短信驗證碼）
- **生物識別**（指紋、面容識別）

## 6. 日誌記錄和審計

### 6.1 審計日誌表

```sql
CREATE TABLE data_processing_logs (
  id UUID PRIMARY KEY,
  user_id UUID,
  operation_type TEXT,  -- 'read', 'create', 'update', 'delete'
  resource_type TEXT,   -- 'transaction', 'location', etc
  resource_id UUID,
  ip_address INET,
  user_agent TEXT,
  created_at TIMESTAMP WITH TIME ZONE
);
```

### 6.2 日誌記錄規則

記錄以下操作：

| 操作 | 記錄內容 | 保留期 |
|------|---------|--------|
| **數據訪問** | 用戶 ID、資源、時間戳 | 90 天 |
| **數據修改** | 用戶 ID、舊值、新值、時間戳 | 2 年 |
| **身份認證** | 用戶、時間戳、成功/失敗 | 30 天 |
| **系統事件** | 事件類型、詳情、時間戳 | 1 年 |

### 6.3 敏感信息日誌脫敏

日誌中不應包含：

- 完整的電郵地址（記錄部分如 `user****@example.com`）
- 密碼或密鑰
- 完整的 IP 地址（如果在歐盟，根據 GDPR 要求）
- 完整的信用卡號碼
- 位置的精確座標（可以記錄粗略位置）

## 7. 網絡安全

### 7.1 防火牆規則

```
入站規則：
  - 允許 HTTPS (443) 來自任何地方
  - 允許 HTTP (80)，重定向到 HTTPS
  - 拒絕所有其他入站流量

出站規則：
  - 允許 HTTPS 到 API 提供商（Supabase 等）
  - 允許 DNS (53)
  - 拒絕其他不必要的出站連接
```

### 7.2 DDoS 防護

- **CDN**：Supabase 使用 Cloudflare CDN（內置 DDoS 防護）
- **速率限制**：應用級別的速率限制
- **驗證碼**：異常流量的驗證碼挑戰

### 7.3 Web 應用防火牆 (WAF)

啟用 WAF 規則防止：

- XSS（跨站指令碼攻擊）
- SQL 注入
- 路徑遍歷攻擊
- 文件上傳攻擊

## 8. API 安全

### 8.1 認證

所有 API 端點（除登錄/註冊外）需要有效的 JWT 令牌：

```typescript
// 標準 Authorization 標題
Authorization: Bearer <jwt_token>
```

### 8.2 授權

基於 RLS 的授權：

```typescript
// 自動檢查 - 用戶只能訪問自己的數據
const { data } = await supabase
  .from('transactions')
  .select('*')
  // RLS 政策自動過濾到當前用戶
```

### 8.3 速率限制

API 終點受速率限制保護，防止濫用：

```
標準終點：100 請求/小時
身份認證終點：10 請求/分鐘
資料導出終點：1 請求/24 小時
```

### 8.4 輸入驗證

所有 API 輸入都經過驗證：

```typescript
// 驗證規則示例
const transactionSchema = {
  amount: { type: 'number', min: 0.01, max: 999999 },
  category: { type: 'string', enum: ['food', 'transport', ...] },
  merchant_name: { type: 'string', maxLength: 255 },
};
```

## 9. 數據隱私

### 9.1 資料最小化

應用只收集必要的數據：

- ✓ 必須收集：姓名、電郵、交易信息
- ✗ 不收集：社交媒體資訊、瀏覽歷史、位置（除非啟用）

### 9.2 數據匿名化

當不再需要個人身份時，應刪除或匿名化個人數據。

**匿名化流程**：

```sql
-- 帳戶刪除後，執行：
UPDATE users SET
  email = NULL,
  phone_number = NULL,
  display_name = 'deleted_user_' || id,
  deleted_at = CURRENT_TIMESTAMP
WHERE id = user_id;

-- 保留交易但移除用戶關聯
UPDATE transactions SET
  user_id = NULL
WHERE user_id = user_id_to_delete;
```

### 9.3 用戶同意管理

在隱私同意表中追蹤用戶同意：

```sql
CREATE TABLE privacy_consents (
  user_id UUID,
  location_tracking_agreed BOOLEAN,
  photo_analysis_agreed BOOLEAN,
  push_notifications_agreed BOOLEAN,
  agreed_at TIMESTAMP,
  expires_at TIMESTAMP
);
```

### 9.4 用戶權利

用戶可以行使以下權利：

| 權利 | 實現方式 | 時限 |
|------|---------|------|
| **訪問** | `GET /api/user/data-export` | 30 天 |
| **更正** | `PATCH /api/user/profile` | 實時 |
| **刪除** | `DELETE /api/user/account` | 30 天內完全刪除 |
| **可攜** | `GET /api/user/data-export` | 30 天 |

### 10.2 敏感信息處理

在代碼中使用敏感信息時：

```dart
// 不好 - 敏感信息記錄
print('User email: $userEmail');

// 好 - 或者根本不記錄
// 或者只記錄部分: print('User email: ${userEmail.replaceAll(RegExp(r'(.{2})(.*)(.{2})'), r'$1***$3')}');
```

### 10.3 第三方庫安全

- **SBOM**（軟體物料清單）：維護所有依賴的清單
- **漏洞掃描**：使用 Dependabot 或類似工具
- **供應鏈安全**：檢查包簽名和發行人驗證
