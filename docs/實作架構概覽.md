# VoiceLedger 後端實作架構概覽

## 專案概述

**VoiceLedger** (語記) 是一款 AI 財務秘書應用，採用 **Supabase Edge Functions** 實現全無伺服器架構。所有 API 端點均使用 Deno + TypeScript 開發，實現智能記帳、分析和對話功能。

---

## 技術棧

| 層級 | 技術 |
|------|------|
| **計算** | Supabase Edge Functions (Deno/TypeScript) |
| **資料庫** | PostgreSQL (via Supabase) |
| **認證** | Supabase Auth (JWT) |
| **AI 模型** | Claude 3.5 Sonnet (Anthropic API) |
| **外部服務** | Google Places API, RevenueCat |

## 核心 Edge Functions

### 1. Voice Entry (`voice-entry`)

**路由**: `POST /voice-entry`

**核心流程**:
```
使用者語音輸入
    ↓
Whisper 轉錄為文字
    ↓
Edge Function 接收
    ↓
Claude 解析 (LLM)
    ├─ 句子分割
    ├─ 金額抽取
    ├─ 分類識別
    └─ 多筆交易拆分
    ↓
信心分數計算
    ↓
資料庫儲存
    ↓
JSON 回應
```

**關鍵功能**:
- 自動多筆拆分 (例: "咖啡50元，便當100元" → 2筆交易)
- 商家識別與分類映射
- 時間估計 (從描述中提取時間)
- 置信度評分 (0-1)

**使用的 LLM Prompt 範式**:
- Structured JSON 輸出
- 類別白名單驗證
- 金額驗證規則

**資料庫操作**:
```sql
INSERT INTO transactions (
  user_id, amount, currency, category, merchant,
  description, timestamp, source, confidence,
  tags, status, created_at
)
VALUES (...)
```

### 2. Passive Inference (`passive-inference`)

**路由**: `POST /passive-inference`

**核心流程**:
```
多模態線索收集
    ├─ GPS 座標
    ├─ 銀行通知文字
    └─ 照片 (Base64)
    ↓
信號融合
    ├─ GPS 定位匹配 (Google Places)
    ├─ 通知金額提取 (Regex)
    └─ 圖像識別 (Claude Vision)
    ↓
信心分數計算
    ├─ location_match: 0-1
    ├─ notification_match: 0-1
    └─ image_recognition: 0-1
    ↓
交易推理
    ├─ 商家確定
    ├─ 分類推測
    ├─ 金額驗證
    └─ 需要確認判斷
    ↓
待確認列表儲存
    ↓
202/200 回應
```

**信心計分邏輯**:
```
confidence = (location_score + notification_score + image_score) / 3
requires_confirmation = confidence < 0.8
```

**地理位置匹配**:
- Haversine 公式計算距離
- 5 倍 GPS 精度範圍內搜尋
- Google Places API 查詢

**通知解析 Regex 模式**:
```
金額: /[\$￥NT\$]\s*(\d+(?:[.,]\d+)?)/
商家: /店(?:\s*|：)([^金額]*)/
```

### 3. AI Chat (`ai-chat`)

**路由**: `POST /ai-chat`

**核心流程**:
```
使用者問題
    ↓
意圖辨識 (Claude)
    ├─ 識別查詢類型
    ├─ 提取實體 (category, period, etc.)
    └─ 判斷 SQL 範本
    ↓
SQL 生成
    ├─ category_summary
    ├─ trend_analysis
    ├─ balance_query
    ├─ budget_check
    ├─ comparison
    ├─ anomaly_detection
    └─ recommendation
    ↓
資料庫查詢
    ├─ Postgres 執行
    └─ 結果聚合
    ↓
自然語言生成 (Claude)
    ├─ 資料轉換為文字
    ├─ 添加洞察和建議
    └─ 本地化 (繁體中文)
    ↓
JSON 回應 (含資料)
```

**支援的查詢類型**:

| 類型 | 描述 | 示例 |
|------|------|------|
| `balance_query` | 查詢總額/交易數 | "這週花了多少?" |
| `category_summary` | 分類統計 | "食物上花多少?" |
| `trend_analysis` | 趨勢分析 | "最近消費怎樣?" |
| `budget_check` | 預算檢查 | "還剩多少預算?" |
| `comparison` | 時期對比 | "和上週比?" |
| `anomaly_detection` | 異常偵測 | "有沒有奇怪的消費?" |
| `recommendation` | 建議 | "我應該減少什麼?" |

**SQL 範本範例**:

```typescript
// category_summary
SELECT
  category,
  SUM(amount) as total,
  COUNT(*) as transaction_count,
  AVG(amount) as avg_amount
FROM transactions
WHERE user_id = $1
  AND category = $2
  AND timestamp BETWEEN $3 AND $4
  AND status = 'confirmed'
GROUP BY category

// trend_analysis
SELECT
  DATE(timestamp) as date,
  SUM(amount) as daily_total,
  category
FROM transactions
WHERE user_id = $1
  AND timestamp BETWEEN $2 AND $3
  AND status = 'confirmed'
GROUP BY DATE(timestamp), category
ORDER BY date DESC
```

**文本到 SQL 流程**:
1. 使用 Claude 辨識意圖 → SQL 類型
2. 提取時間實體 → date range
3. 使用預先定義的 SQL 範本
4. 動態填入參數

### 4. Daily Wisdom (`daily-wisdom`)

**路由**: `GET /daily-wisdom?date=YYYY-MM-DD&tone=encouraging|cautious|analytical`

**核心流程**:
```
用戶授權查詢
    ↓
檢查快取
    └─ 若存在 → 返回
    ↓
取得過去 7 天交易資料
    ├─ 總支出
    ├─ 每日統計
    └─ 分類明細
    ↓
數據分析
    ├─ 支出趨勢 (增/減/穩)
    ├─ 週末 vs 平日對比
    ├─ 最大分類檢測
    └─ 模式識別
    ↓
Claude 生成金句
    ├─ 根據 tone 調整語氣
    ├─ 引用具體數據
    └─ 包含可行建議
    ↓
非同步儲存
    ↓
返回 (金句 + 歷史記錄)
```

**數據分析邏輯**:

```typescript
// 支出趨勢判斷
first_half_avg = 前半週平均
second_half_avg = 後半週平均

if (second_half_avg > first_half_avg * 1.1) trend = "increasing"
else if (second_half_avg < first_half_avg * 0.9) trend = "decreasing"
else trend = "stable"

// 週末比例計算
weekday_avg = Mon-Fri 平均
weekend_avg = Sat-Sun 平均
ratio = weekend_avg / weekday_avg
```

**金句生成 Prompt**:
```
使用者消費數據統計
    ↓
根據 tone (encouraging/cautious/analytical) 調整
    ↓
生成 2-3 句意義深遠的金句
    ├─ 引用具體數字
    ├─ 加入比喻或類比
    └─ 使用繁體中文
    ↓
配套一個可行建議
```

**快取策略**:
- 按用戶 + 日期為 Key
- 同日期檢查時直接返回
- 隔日自動失效

## 資料庫架構

### transactions 表
```sql
CREATE TABLE transactions (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID NOT NULL REFERENCES auth.users(id),

  -- 基本資訊
  amount NUMERIC(10,2) NOT NULL,
  currency VARCHAR(3) DEFAULT 'TWD',
  category VARCHAR(50) NOT NULL,
  merchant VARCHAR(255),
  description TEXT,

  -- 時間與來源
  timestamp TIMESTAMPTZ NOT NULL,
  source VARCHAR(20) NOT NULL, -- 'voice'|'passive'|'manual'|'import'

  -- 質量指標
  confidence NUMERIC(3,2), -- 0.00-1.00
  status VARCHAR(20) DEFAULT 'pending', -- 'pending'|'confirmed'|'flagged'

  -- 額外資訊
  tags TEXT[] DEFAULT '{}',
  notes TEXT,
  receipt_image_url VARCHAR(500),
  location GEOGRAPHY(POINT),
  metadata JSONB,

  -- 審計
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),

  -- 索引
  UNIQUE(user_id, timestamp, merchant, amount) -- 防重複
);

CREATE INDEX idx_transactions_user_date ON transactions(user_id, timestamp DESC);
CREATE INDEX idx_transactions_category ON transactions(user_id, category);
CREATE INDEX idx_transactions_status ON transactions(user_id, status);
```

### daily_wisdom 表
```sql
CREATE TABLE daily_wisdom (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID NOT NULL REFERENCES auth.users(id),

  date DATE NOT NULL,
  quote TEXT NOT NULL,
  tone VARCHAR(20),

  -- 數據洞察
  data_insight JSONB,
  actionable_tip TEXT,

  -- 生成時間
  generated_at TIMESTAMPTZ DEFAULT NOW(),
  expires_at TIMESTAMPTZ,

  UNIQUE(user_id, date)
);
```

## 錯誤處理標準

所有端點遵循統一格式:
```json
{
  "success": false,
  "error": "<ERROR_CODE>",
  "message": "<human_readable_message>",
  "code": "<SUBTYPE>",
  "details": {}
}
```

### 常見錯誤碼
| 碼 | 含義 | HTTP 狀態 |
|----|------|----------|
| `AUTH_ERROR` | 認證失敗 | 401 |
| `INVALID_INPUT` | 輸入驗證失敗 | 400 |
| `PARSE_ERROR` | LLM 解析失敗 | 500 |
| `DB_ERROR` | 資料庫操作失敗 | 500 |
| `INFERENCE_ERROR` | 推理失敗 | 500 |

## 安全性考慮

### 1. 輸入驗證
- 文字長度限制 (max 1000 chars)
- Base64 圖像大小限制
- SQL 注入預防 (參數化查詢)

### 2. 敏感資訊
- 不在日誌中記錄金額
- 通知文本 (可能包含私密信息) 立即銷毀
- 圖像在處理後刪除

### 3. 速率限制
```
按用戶 ID + IP 組合限制
違反時返回 429 Too Many Requests
```
