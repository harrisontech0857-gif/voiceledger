# VoiceLedger 專案交付物清單

## 概述

VoiceLedger (語記) 是一個完整的 AI 財務秘書應用後端架構設計，包含 8 個核心 API 端點和 4 個 Edge Functions 實現。

**技術棧**: Supabase Edge Functions + Deno/TypeScript + Claude 3.5 Sonnet

**開發日期**: 2026-03-18

### 2. Edge Functions 實現

#### Edge Function 1: `voice-entry` (`backend/supabase/functions/voice-entry/index.ts`)
**功能** (380 行代碼):
- 接收 Whisper 轉錄文字
- Claude LLM 語意解析
- 多筆交易自動拆分
- 自動分類和商家識別
- 時間估計演算法
- 信心分數計算
- PostgreSQL 資料庫儲存

**核心邏輯**:
```
Whisper 轉錄 → Claude 解析 → 類別映射 → 信心評分 → DB 儲存 → JSON 回應
```

**支援的分類**: 25+ (食物、交通、娛樂、購物等)

---

#### Edge Function 2: `passive-inference` (`backend/supabase/functions/passive-inference/index.ts`)
**功能** (450 行代碼):
- GPS 座標定位匹配 (Haversine 距離計算)
- 銀行通知文字解析 (Regex 模式)
- 照片識別 (Claude Vision API)
- 多信號融合
- 信心分數融合 (加權平均)
- 待確認交易管理

**信號融合演算法**:
```
location_score (0-1) ⟶┐
notification_score (0-1)├→ average → confidence
image_score (0-1) ────┘
```

**地理位置匹配**:
- Haversine 公式 (地球距離計算)
- 5 倍 GPS 精度範圍搜尋
- Mock Google Places API 集成點

---

#### Edge Function 3: `ai-chat` (`backend/supabase/functions/ai-chat/index.ts`)
**功能** (400 行代碼):
- 自然語言意圖識別
- Text-to-SQL 查詢生成
- 7 種查詢類型支援
- 資料庫動態查詢
- 自然語言回應生成
- 繁體中文本地化

**支援的查詢類型**:
1. `balance_query` — 餘額查詢
2. `category_summary` — 分類統計
3. `trend_analysis` — 趨勢分析
4. `budget_check` — 預算檢查
5. `comparison` — 時期對比
6. `anomaly_detection` — 異常偵測
7. `recommendation` — 理財建議

**SQL 範本**: 7 個預定義範本，支持動態參數填充

---

#### Edge Function 4: `daily-wisdom` (`backend/supabase/functions/daily-wisdom/index.ts`)
**功能** (380 行代碼):
- 過去 7 天消費數據分析
- 支出趨勢判斷 (增/減/穩)
- 週末 vs 平日對比分析
- 個人化金句生成 (Claude)
- 可行建議推薦
- 快取策略 (按日期)
- 歷史記錄管理

**分析指標**:
- 總支出、日均支出
- 交易計數、最大分類
- 支出趨勢方向
- 消費模式識別

**金句類型**: 3 種語氣 (encouraging, cautious, analytical)

## 🚀 快速開始

### 1. 環境設置 (5 分鐘)
```bash
supabase init
npm install -g supabase
supabase start  # 本地開發
```

### 2. 複製 Edge Functions (1 分鐘)
```bash
cp backend/supabase/functions/* <your-project>/supabase/functions/
```

### 3. 設置環境變數 (2 分鐘)
```bash
supabase secrets set ANTHROPIC_API_KEY "sk-ant-..."
supabase secrets set GOOGLE_PLACES_API_KEY "AIza..."
```

### 4. 初始化資料庫 (3 分鐘)
```bash
supabase db reset
```

### 5. 部署到生產 (2 分鐘)
```bash
supabase link --project-ref <ref>
supabase functions deploy
```

**總耗時**: ~13 分鐘

## ✨ 設計原則

1. **無伺服器優先** — 所有邏輯運行於 Edge Functions
2. **LLM 驅動** — 複雜邏輯由 Claude 處理
3. **多信號融合** — 多個數據源綜合判斷
4. **用戶隱私優先** — RLS 嚴格控制資料訪問
5. **易於擴展** — 模塊化設計便於新增功能
6. **本地化優先** — 繁體中文最佳化
