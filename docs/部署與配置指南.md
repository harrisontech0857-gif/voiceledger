## 環境設置

### 前置需求
- Node.js 18+ (or Deno 1.40+)
- Supabase CLI (`npm install -g supabase`)
- Anthropic API Key
- Google Cloud API Key (Places API)

```bash
npm install -g supabase

```

## Supabase 配置

### 1. 創建 Supabase 專案

前往 [supabase.com](https://supabase.com):
1. 點選「New Project」
2. 輸入專案名稱: `voiceledger`
3. 設置資料庫密碼
4. 選擇區域 (建議 Asia - Singapore 靠近用戶)
5. 等待初始化完成 (約 5 分鐘)

### 2. 複製連接資訊

在 Supabase Dashboard 的「Settings > API」取得:
```
SUPABASE_URL=https://<project-id>.supabase.co
SUPABASE_ANON_KEY=<anon-key>
SUPABASE_SERVICE_KEY=<service-key>
```

### 3. 啟用必要的 Extension

在 Supabase SQL Editor 執行:
```sql
-- 啟用 UUID
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- 啟用 PostGIS (地理位置支持)
CREATE EXTENSION IF NOT EXISTS "postgis";

-- 啟用 pgcrypto (加密)
CREATE EXTENSION IF NOT EXISTS "pgcrypto";
```

### 4. 配置 Row Level Security (RLS)

在 Supabase SQL Editor 執行:
```sql
-- 為 transactions 表啟用 RLS
ALTER TABLE transactions ENABLE ROW LEVEL SECURITY;

-- 建立策略: 用戶只能看自己的交易
CREATE POLICY "Users can view own transactions"
  ON transactions FOR SELECT
  USING (auth.uid() = user_id);

-- 建立策略: 用戶只能插入自己的交易
CREATE POLICY "Users can insert own transactions"
  ON transactions FOR INSERT
  WITH CHECK (auth.uid() = user_id);

-- 建立策略: 用戶只能更新自己的交易
CREATE POLICY "Users can update own transactions"
  ON transactions FOR UPDATE
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);

-- 為 daily_wisdom 表啟用 RLS
ALTER TABLE daily_wisdom ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view own wisdom"
  ON daily_wisdom FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "Users can insert own wisdom"
  ON daily_wisdom FOR INSERT
  WITH CHECK (auth.uid() = user_id);
```

## Edge Functions 部署

### 1. 本地開發與測試

#### 啟動 Supabase 本地環境
```bash
supabase start

# 輸出範例:
# Started Supabase local development server
# Supabase URL: http://localhost:54321
# anon key: eyJh...
```

#### 測試 Voice Entry 函數
```bash
curl -X POST http://localhost:54321/functions/v1/voice-entry \
  -H "Authorization: Bearer <jwt_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "transcript": "早上咖啡五十塊",
    "audio_duration_ms": 3000,
    "timestamp": "2026-03-18T09:30:00Z",
    "metadata": {
      "device_id": "device-123",
      "language": "zh-TW"
    }
  }'
```

#### 獲取有效的 JWT Token (本地測試)
```bash
# 在 Supabase 本地環境建立測試用戶
curl -X POST http://localhost:54321/auth/v1/signup \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "Test@12345"
  }'

# 登入獲取 token
curl -X POST http://localhost:54321/auth/v1/token?grant_type=password \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "Test@12345"
  }'
```

### 2. 部署到生產環境

#### 鏈接遠端 Supabase 專案
```bash
supabase link --project-ref <project-ref>

# 當提示輸入密碼時，輸入您的資料庫密碼
```

#### 部署 Edge Functions
```bash
# 部署所有函數
supabase functions deploy

# 或部署特定函數
supabase functions deploy voice-entry
supabase functions deploy passive-inference
supabase functions deploy ai-chat
supabase functions deploy daily-wisdom
```

#### 驗證部署
```bash
# 查看已部署的函數
supabase functions list

# 查看函數日誌
supabase functions logs voice-entry
```

在 Supabase Dashboard「Settings > Secrets」設置:

```bash
supabase secrets set ANTHROPIC_API_KEY "sk-ant-v0-..."

supabase secrets set REVENUCAT_API_KEY "<key>"
```

```bash
# 列出所有 secrets
supabase secrets list

# 在 Edge Function 中驗證
supabase functions deploy --verify
```

## 資料庫初始化

### 1. 建立遷移文件

創建 `supabase/migrations/20260318_init_tables.sql`:

```sql
-- 建立 transactions 表
CREATE TABLE transactions (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,

  -- 基本資訊
  amount NUMERIC(10,2) NOT NULL CHECK (amount > 0),
  currency VARCHAR(3) NOT NULL DEFAULT 'TWD',
  category VARCHAR(50) NOT NULL,
  subcategory VARCHAR(100),
  merchant VARCHAR(255),
  description TEXT,

  -- 時間與來源
  timestamp TIMESTAMPTZ NOT NULL,
  source VARCHAR(20) NOT NULL CHECK (source IN ('voice', 'passive', 'manual', 'import')),

  -- 質量指標
  confidence NUMERIC(3,2) CHECK (confidence >= 0 AND confidence <= 1),
  status VARCHAR(20) NOT NULL DEFAULT 'pending' CHECK (status IN ('pending', 'confirmed', 'flagged')),

  -- 額外資訊
  tags TEXT[] DEFAULT ARRAY[]::TEXT[],
  notes TEXT,
  receipt_image_url VARCHAR(500),
  location GEOGRAPHY(POINT),
  metadata JSONB DEFAULT '{}'::JSONB,

  -- 審計
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 建立索引
CREATE INDEX idx_transactions_user_date ON transactions(user_id, timestamp DESC);
CREATE INDEX idx_transactions_user_category ON transactions(user_id, category, timestamp DESC);
CREATE INDEX idx_transactions_status ON transactions(user_id, status);
CREATE INDEX idx_transactions_created_at ON transactions(created_at DESC);

-- 防重複索引
CREATE UNIQUE INDEX idx_duplicate_check
  ON transactions(user_id, timestamp, merchant, amount)
  WHERE status = 'confirmed';

-- 建立 daily_wisdom 表
CREATE TABLE daily_wisdom (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,

  date DATE NOT NULL,
  quote TEXT NOT NULL,
  tone VARCHAR(20),
  data_insight JSONB,
  actionable_tip TEXT,

  generated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  expires_at TIMESTAMPTZ,

  UNIQUE(user_id, date)
);

CREATE INDEX idx_daily_wisdom_user_date ON daily_wisdom(user_id, date DESC);

-- 建立 subscriptions 表
CREATE TABLE subscriptions (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,

  plan VARCHAR(20) NOT NULL CHECK (plan IN ('free', 'premium', 'plus')),
  status VARCHAR(20) NOT NULL CHECK (status IN ('active', 'cancelled', 'expired')),

  started_at TIMESTAMPTZ NOT NULL,
  expires_at TIMESTAMPTZ,
  auto_renew BOOLEAN DEFAULT true,

  external_id VARCHAR(100), -- RevenueCat subscription ID

  entitlements TEXT[] DEFAULT ARRAY[]::TEXT[],
  metadata JSONB DEFAULT '{}'::JSONB,

  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_subscriptions_user ON subscriptions(user_id);
CREATE INDEX idx_subscriptions_status ON subscriptions(status);

-- 建立 webhook_logs 表 (用於調試)
CREATE TABLE webhook_logs (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  event_type VARCHAR(50) NOT NULL,
  payload JSONB NOT NULL,
  processed BOOLEAN DEFAULT false,
  error_message TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_webhook_logs_event ON webhook_logs(event_type, created_at DESC);

-- 啟用 RLS
ALTER TABLE transactions ENABLE ROW LEVEL SECURITY;
ALTER TABLE daily_wisdom ENABLE ROW LEVEL SECURITY;
ALTER TABLE subscriptions ENABLE ROW LEVEL SECURITY;

-- 建立 RLS 策略
CREATE POLICY "Users can view own transactions"
  ON transactions FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "Users can insert own transactions"
  ON transactions FOR INSERT
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users can update own transactions"
  ON transactions FOR UPDATE
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users can view own wisdom"
  ON daily_wisdom FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "Users can insert own wisdom"
  ON daily_wisdom FOR INSERT
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users can view own subscriptions"
  ON subscriptions FOR SELECT
  USING (auth.uid() = user_id);

-- 建立更新觸發器 (自動更新 updated_at)
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_transactions_updated_at
  BEFORE UPDATE ON transactions
  FOR EACH ROW
  EXECUTE FUNCTION update_updated_at_column();

CREATE TRIGGER update_subscriptions_updated_at
  BEFORE UPDATE ON subscriptions
  FOR EACH ROW
  EXECUTE FUNCTION update_updated_at_column();
```

## WebHook 配置

### 1. RevenueCat WebHook 設置

在 RevenueCat Dashboard:
1. 進入「Integrations > Webhooks」
2. 新增 WebHook:
   - **URL**: `https://<project-id>.supabase.co/functions/v1/webhooks/revenucat`
   - **Events**: subscription_started, subscription_updated, subscription_cancelled, subscription_expired
   - **Authentication**: 啟用簽名驗證

3. 複製 Webhook Secret，設置為環境變數:
```bash
supabase secrets set REVENUCAT_WEBHOOK_SECRET "secret_..."
```

### 2. 測試 WebHook

```bash
# 從 RevenueCat 發送測試事件
curl -X POST https://<project-id>.supabase.co/functions/v1/webhooks/revenucat \
  -H "Content-Type: application/json" \
  -H "X-RevenueCat-Signature: <signature>" \
  -d '{
    "event": {
      "type": "subscription_started"
    },
    "subscriber": {
      "app_user_id": "user_123"
    }
  }'
```

## 監測與日誌

### 1. 查看 Edge Function 日誌

```bash
# 實時日誌
supabase functions logs voice-entry --follow

# 過濾特定錯誤
supabase functions logs voice-entry | grep ERROR

# 查看特定請求
supabase functions logs voice-entry | grep "X-Request-Id"
```

### 2. 設置日誌告警

在 Supabase Dashboard「Logs」:
1. 建立自定義告警
2. 監測條件: `error` 或 `500`
3. 通知: Email / Slack

### 3. 應用效能監測 (APM)

可選集成:
- **Sentry**: 錯誤追蹤
- **DataDog**: 完整可觀測性
- **New Relic**: 效能監測

### 4. 自定義監測

在每個 Edge Function 中添加:
```typescript
// 發送指標到外部監測服務
const metrics = {
  request_id: generate_uuid(),
  function: "voice-entry",
  processing_time: processing_time,
  success: response.success,
  timestamp: new Date().toISOString()
};

// 發送到 Datadog / Sentry / 其他
await send_metrics(metrics);
```

## 故障排除

### 常見問題

#### 1. 「Invalid JWT」錯誤
```bash
# 檢查 token 有效期
# 檢查 Authorization header 格式
# 確認 Supabase URL 正確
```

#### 2. 「CORS error」
在 Supabase SQL Editor:
```sql
-- 檢查 CORS 設置
SELECT * FROM pg_settings WHERE name LIKE '%cors%';
```

#### 3. 「函數超時」
增加超時時間:
```toml
# supabase/config.toml
[functions]
  [functions."voice-entry"]
  verify_jwt = true
  timeout = 60  # 秒
```

#### 4. 資料庫連接失敗
```bash
# 檢查 Supabase URL
echo $SUPABASE_URL

# 測試連接
psql "postgresql://postgres:password@host:5432/postgres"
```
