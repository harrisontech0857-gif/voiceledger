## 設計高級特性

### 🔒 安全與隱私
- **Row Level Security (RLS)**: 12 個政策覆蓋所有用戶數據
- **數據隔離**: 用戶只能訪問自己和有權限的數據
- **審計日誌**: 所有交易變更都有記錄
- **軟刪除**: 支持數據恢復和 GDPR 合規

### ⚡ 性能優化
- **複合索引**: 針對最常見查詢模式
- **部分索引**: 用於未處理的記錄
- **全文搜尋**: GIN 索引支持商家和備註搜尋
- **觸發器**: 預計算統計信息，避免應用層計算

### 🤖 AI 友好設計
- **JSONB 靈活性**: 支持動態數據結構
- **分析表**: 預計算的行為分析結果
- **信心分數**: 用於 AI 建議質量評估
- **對話歷史**: 支持上下文連續性

### 📊 數據驅動
- **三個視圖**: 用戶月度統計、分類支出、預算利用率
- **統計信息**: 自動維護用戶和商家統計
- **趨勢分析**: 支持消費趨勢檢測
- **異常檢測**: 支持異常交易識別

### 🏠 家庭友好
- **多帳本**: 支持共享家庭、旅遊、項目分開記帳
- **靈活權限**: 4 種角色 (owner/admin/member/viewer)
- **自動結算**: 計算每個人應支付金額
- **隱私控制**: 可配置可見性設置

## 使用指南

### 部署步驟

1. **準備**
   ```bash
   cd /backend/supabase/
   # 確保有 Supabase 項目和正確的連接設置
   ```

2. **應用遷移** (3 種方式)
   - **Supabase CLI**: `supabase db push`
   - **Supabase Dashboard**: 複製粘貼 SQL
   - **psql 命令行**: `psql < migrations/001_初始架構.sql`

3. **驗證** (執行檢查查詢)
   ```sql
   SELECT COUNT(*) FROM information_schema.tables
   WHERE table_schema = 'public';
   -- 應返回 27
   ```

4. **監控**
   - 設置備份策略
   - 配置監控告警
   - 定期檢查查詢性能

### 常見任務

#### 創建新用戶
```sql
INSERT INTO users (email, display_name)
VALUES ('user@example.com', 'John Doe')
RETURNING id;
```

#### 記錄交易
```sql
INSERT INTO transactions (
  user_id, amount, category, source_type,
  transaction_date, confidence_score
) VALUES (?, ?, ?, ?, ?, ?)
RETURNING id;
```

#### 查詢月度統計
```sql
SELECT * FROM user_monthly_stats
WHERE user_id = ?
ORDER BY month DESC;
```

#### 檢查預算狀態
```sql
SELECT * FROM budget_utilization
WHERE user_id = ?
AND status IN ('warning', 'exceeded');
```
